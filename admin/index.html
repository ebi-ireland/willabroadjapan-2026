📖 大学スコアとは

  • ターゲットスコア：ユーザーの入力から計算した得点がこの数値の何%かで合否を判定します
  • 合格判定：得点 ÷ ターゲットスコア ≥ 合格しきい値(%) → 合格圏、≥ 可能性しきい値 → 可能性あり
  • Need-based対応：チェックすると「Need-based Scholarship希望」のユーザーでも減点なし
  • 大量登録は 📥 インポートタブ をご利用ください

📖 インポート形式

  • 1行1大学で貼り付けてください
  • 最小形式（大学名とスコアのみ）：Harvard University,16000
  • 標準形式：Harvard University,16000,アメリカ,0
  • 列順：①大学名, ②スコア, ③国（省略可）, ④Need-based 0か1（省略可）
  • スコアを省略または0にすると 10000 が自動設定されます
  • 重複チェック：すでに同名の大学が存在する場合はスキップされます
  • 区切り文字はカンマ・タブ・スペースから選べます

📖 採点テーブルの仕組み

  • ユーザーが入力したスコアが「最小値 ≤ スコア ≤ 最大値」の範囲内に入ったとき、そのptsが加算されます
  • 同じ項目（GPA等）で複数のブラケットを設定することで段階的な加点が可能です
  • クラスランクは数値範囲ではなくキー値（cr_5等）で一致させます
  • 各カテゴリの加点上限は「🧮 計算調整」タブで設定できます

📖 計算調整の仕組み

  • 重み係数（Weight）：各カテゴリの採点テーブルで得たptsにこの係数を掛けます。
    例：GPA重み=1.5 なら、GPA採点結果×1.5が実際の加算点になります
  • 最大pts上限（Cap）：各カテゴリから得られるptsの上限。0は上限なし。
    例：英語資格上限=2000 なら、英語スコアが高くても最大2000ptsまで
  • 変更後は診断ページをリロードすると即反映されます
  • 初期値は全て重み1.0、上限なし（0）です

📖 全体設定の意味

  • 合格判定しきい値：ユーザー得点 ÷ ターゲットスコア がこの値(%)以上で「合格圏」表示
  • 可能性ありしきい値：上記未満・この値以上で「可能性あり」表示。それ未満は「厳しい」
  • 最大選択大学数：ユーザーが一度に診断できる大学数の上限

🔍 SEO管理
各ページのタイトル・description・OGP・Robotsを設定します

📖 使い方ガイド

  • ページパス：URLパスを入力します（例: /student-japan.html）
  • タイトル：ブラウザのタブに表示されるタイトル（60文字以内推奨）
  • ディスクリプション：Google検索結果に表示される説明文（120文字以内推奨）

📅 公式締め切り管理
ユーザーの締め切りリストページに表示される「公式情報」を管理します

  • 公式締め切りとは：管理者が登録した締め切り情報で、全ユーザーの「締め切りリスト」ページに共通表示されます

⚠️ エラーログ
サーバーで発生したエラー・警告の記録を確認します

📖 使い方ガイド

  • error：サーバーがクラッシュしそうな重大なエラー。すぐに対処が必要
  • warn：軽微な異常（認証失敗が多いなど）。念のため確認推奨
  • info：通常の動作ログ（バックアップ完了など）
  • 「詳細」ボタン：スタックトレース（エラーの詳しい発生場所）を確認できます
  • 定期削除：ログは増え続けるため、月1回「30日以上前を削除」を実行してDBを軽くしてください
  • 同じIPが大量に出る場合：DoS攻撃の可能性あり。該当IPをファイアウォールでブロック検討

💾 データベースバックアップ
MySQLデータベース全体の自動・手動バックアップを管理します

📖 使い方ガイド

  • 自動バックアップ：毎日午前3時（サーバー時間）に自動で実行されます。何もしなくてOKです
  • 手動バックアップ：大規模なデータ更新の前後、または不安なときにいつでも実行できます
  • 保存場所：サーバー内の /backups/ フォルダに .sql ファイルとして保存
  • 保管期間：90日分が自動保持。91日以上前のファイルは自動削除されます
  • リストア方法：万が一データが消えた場合は、mysql -u root -p データベース名 < backup_ファイル名.sql で復元できます
  • 推奨：月1回、最新の .sql ファイルをPCにダウンロードしてローカルにも保存してください

手動バックアップ実行

ボタンを押すとサーバー上で mysqldump が実行され、 全テーブルのスナップショットが保存されます。
データ量によっては数秒〜数十秒かかる場合があります。
